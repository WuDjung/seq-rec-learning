

# SASRec 论文精读

**作者**: Wang-Cheng Kang, Julian McAuley (UC San Diego)

**发表**: ICDM 2018 (Long Paper)

**引用量**: 这是序列推荐领域引用最高的论文之一

## 第一部分：摘要和引言——这篇论文要解决什么问题？

### 论文的核心论点

作者观察到，当时序列推荐领域有两大阵营：

**马尔可夫链（MC）阵营**：只看最近一个或几个行为来预测下一个。模型简单，参数少，在**稀疏数据集**上表现好（因为不容易过拟合）。

**RNN 阵营**：用隐状态记住整个历史。模型复杂，参数多，在**稠密数据集**上表现好（因为有足够的数据来训练复杂模型）。

作者的核心洞察是：**能不能有一个模型，既能看到全部历史（像 RNN），又能自适应地只关注少数关键行为（像 MC）？** 这就是自注意力机制天然能做到的事情——它可以从任意长的序列中，自动学习"应该关注哪几个位置"。

这个定位非常精准，也是论文能发表在顶会的关键原因之一：它不只是把 Transformer 搬到推荐领域，而是**清晰地论证了为什么自注意力恰好能解决序列推荐中 MC 和 RNN 之间的矛盾**。

### 对你写论文的启发

一篇好论文的引言不是"我用了新技术 X，效果好"，而是"领域中存在一个具体的矛盾/困境，我的方法恰好能解决这个矛盾"。记住这个模式，写论文时可以这样构建动机。

## 第二部分：相关工作（Related Work）

论文的相关工作分了四个小节，覆盖了整个推荐系统的技术谱系。对你来说，重点关注以下对应关系：

| 论文中的类别 | 代表方法 | 核心思想 |
|---|---|---|
| 通用推荐 | MF, BPR, NeuMF | 不考虑时序，只建模 user-item 匹配 |
| 时序推荐 | TimeSVD++ | 显式建模时间戳本身（几月几号） |
| 序列推荐 (MC) | FPMC, Caser | 建模行为的先后顺序（第1步→第2步） |
| 序列推荐 (RNN) | GRU4Rec | 用隐状态记忆整个序列 |
| 注意力机制 | Transformer | 自注意力，不依赖 RNN/CNN |

**时序推荐 vs 序列推荐**的区别很重要：时序推荐关心的是"几月几号做了什么"（绝对时间），序列推荐关心的是"先做了 A 后做了 B"（相对顺序）。SASRec 属于后者，所以它的位置编码表示的是"第几个行为"，而不是"具体时间戳"。

## 第三部分：方法论（Methodology）——核心内容

这是论文最重要的部分，我逐个组件讲，并对照你的代码。

### 3.1 Embedding Layer

**论文原文思路**：将用户的行为序列转化为固定长度 $n$（max_len），超长截断、不足左侧填充。每个物品用一个 $d$ 维的嵌入向量表示，再加上可学习的位置嵌入。

$$\hat{E} = \begin{bmatrix} M_{s_1} + P_1 \\ M_{s_2} + P_2 \\ \vdots \\ M_{s_n} + P_n \end{bmatrix}$$

其中 $M$ 是物品嵌入矩阵，$P$ 是位置嵌入矩阵。

**对应复现代码**（`model.py` 中的 `forward` 方法）：

```python
x = self.item_embedding(input_seq) + self.position_embedding(positions)
```

完全一致。论文还提到他们试过 Transformer 原文中的固定位置编码（正弦/余弦），但效果不如可学习的位置嵌入。这在推荐场景中是合理的——位置的语义在推荐中和在 NLP 中不同，值得让模型自己去学。

### 3.2 Self-Attention Block

论文中一个 Self-Attention Block 包含两个子层：

**子层一：自注意力层**

$$S = \text{SA}(\hat{E}) = \text{Attention}(\hat{E}W^Q, \hat{E}W^K, \hat{E}W^V)$$

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d}}\right) V$$

**对应复现代码**：这部分由 `nn.TransformerEncoderLayer` 内部实现。不需要手写注意力计算，PyTorch 已经封装好了。

**因果性（Causality）**——这是 SASRec 的关键设计。论文明确指出：预测第 $t+1$ 个物品时，只能看到位置 $1$ 到 $t$ 的信息，必须通过 mask 禁止看到未来。

**对应复现代码**：

```python
causal_mask = torch.triu(
    torch.ones(seq_len, seq_len, device=input_seq.device),
    diagonal=1
).bool()
```

这就是因果 mask。上三角矩阵，确保位置 $i$ 只能关注位置 $\leq i$ 的物品。

**子层二：逐位置前馈网络（Point-Wise FFN）**

$$F_i = \text{ReLU}(S_i W^{(1)} + b^{(1)}) W^{(2)} + b^{(2)}$$

这一层的作用是引入**非线性**。自注意力虽然能自适应地加权聚合历史物品，但本质上是线性运算。FFN 给模型增加了非线性变换能力。

**对应你的代码**：同样由 `nn.TransformerEncoderLayer` 内部实现，其中 `dim_feedforward=embed_dim * 4` 就是 FFN 的隐藏层维度。

### 3.3 堆叠自注意力块 + 正则化

论文提出堆叠多个 Self-Attention Block，并使用三种正则化技术：

**残差连接（Residual Connection）**：$g(x) = x + \text{SubLayer}(x)$。作者给了一个很好的直觉：在序列推荐中，最后访问的物品对预测下一个物品通常最重要。经过多层 Transformer 后，这个信息可能被稀释。残差连接允许底层信息直接传到高层。

**Layer Normalization**：归一化每个样本的特征向量到零均值单位方差，稳定训练。

**Dropout**：随机关闭神经元，防止过拟合。

论文中使用的是 **Post-Norm**（先做子层计算，再加残差，最后 LayerNorm）：

$$g(x) = x + \text{Dropout}(g(\text{LayerNorm}(x)))$$

**对应复现代码**：复现代码的是 `norm_first=True`，即 **Pre-Norm**（先 LayerNorm，再做子层计算，最后加残差）。这和原文略有不同，但 Pre-Norm 在实践中通常训练更稳定，是更现代的做法。这属于合理的实现差异。

### 3.4 预测层

$$r_{i,t} = F_t^{(b)} M_i^T$$

即用第 $b$ 层（最后一层）Transformer 在位置 $t$ 的输出向量，与物品 $i$ 的嵌入做内积。

**Shared Item Embedding（共享物品嵌入）**：论文的一个重要设计决策。预测层使用的物品嵌入 $M$ 和输入端的物品嵌入是**同一个**。论文论证了这样做不会丧失表达能力——因为 FFN 的非线性变换可以让 $\text{FFN}(M_i)M_j^T \neq \text{FFN}(M_j)M_i^T$，从而表达不对称的物品转移关系。

**对应复现代码**（`predict` 方法）：

```python
all_item_emb = self.item_embedding.weight[1:]   # 共享嵌入
scores = torch.matmul(user_repr, all_item_emb.T)
```

完全一致。

**不使用显式用户嵌入**：论文提到他们也试过给每个用户一个独立的嵌入向量，但发现不提升效果。因为模型已经通过处理用户的整个行为序列，隐式地建模了用户偏好。

### 3.5 训练

$$\mathcal{L} = -\sum_{S^u \in S} \sum_{t} \left[ \log \sigma(r_{o_t, t}) + \sum_{j \notin S^u} \log(1 - \sigma(r_{j, t})) \right]$$

论文使用的是 **BCE Loss + 每个时间步随机采一个负样本**。

**对应复现代码**：复现代码用的是 **Cross-Entropy Loss（多分类）**，不需要显式负采样。这是一个合理的改进——近年来的研究表明，Cross-Entropy 通常优于 BCE + 负采样，因为它在每一步考虑了所有物品的对比，信息量更大。当然代价是计算量更大（需要对所有物品计算 logits），但在 MovieLens-1M 这个规模上没问题。

### 3.6 复杂度分析

**空间复杂度**：$O(|I|d + nd + d^2)$，不随用户数增长。这是因为 SASRec 没有用户嵌入矩阵。

**时间复杂度**：$O(n^2 d + n d^2)$，主要是自注意力的 $O(n^2 d)$。但关键优势是可以并行计算（GPU 加速），不像 RNN 必须顺序执行。

### 3.7 与经典模型的关系（非常重要的一节）

论文证明了 SASRec 可以退化为几个经典模型：

**去掉自注意力块 + 去掉位置嵌入 + 使用非共享嵌入 → FMC**（Factorized Markov Chains）

**在上面基础上加用户嵌入 → FPMC**

**使用一层自注意力 + 均匀注意力权重 + 去掉位置嵌入 + 非共享嵌入 → FISM**（Factorized Item Similarity Model）

这说明 SASRec 是这些经典模型的**泛化**。这种"我的方法包含已有方法作为特例"的论证方式在论文写作中非常有力，可以学习这种思路。

## 第四部分：实验

### 数据集和评估方式

论文使用了四个数据集，涵盖从稀疏到稠密：

| 数据集 | 用户数 | 物品数 | 平均序列长度 | 稀疏程度 |
|---|---|---|---|---|
| Amazon Beauty | 52K | 57K | 7.6 | 非常稀疏 |
| Amazon Games | 31K | 24K | 9.3 | 稀疏 |
| Steam | 335K | 13K | 11.0 | 中等 |
| MovieLens-1M | 6K | 3.4K | 163.5 | 稠密 |

评估方式：**Leave-One-Out + 采样评估（100 个负样本 + 1 个正样本）**。这也是为什么实验结果不能和论文结果直接对比，复现代码是全量评估，论文用的是采样评估。

论文中 SASRec 在 ML-1M 上的结果：**HR@10 = 0.8245, NDCG@10 = 0.5905**。注意这是采样评估下的数字。

### 超参数

论文的默认设置和复现代码对比：

| 超参数 | 论文 | 复现代码 | 说明 |
|---|---|---|---|
| 自注意力层数 $b$ | 2 | 2 | 一致 |
| 嵌入维度 $d$ | 50 | 64 | 略有差异，影响不大 |
| Dropout | 0.2 (ML-1M) | 0.2 | 一致 |
| 最大序列长度 $n$ | 200 (ML-1M) | 200 | 一致 |
| 优化器 | Adam, lr=0.001 | Adam, lr=0.001 | 一致 |
| Batch size | 128 | 128 | 一致 |

### 消融实验（Ablation Study）的关键发现

这部分对你理解每个组件的作用非常重要：

**去掉位置嵌入**：在稀疏数据集上反而变好了（因为短序列中位置信息不重要），但在稠密数据集上变差了。

**使用非共享嵌入**：所有数据集上都变差了，说明共享嵌入不仅省参数，还是一种有效的正则化。

**去掉残差连接**：严重掉点（>10%），说明残差连接至关重要。

**去掉 Dropout**：也掉点，尤其在稀疏数据集上。

**层数的影响**：从 0 层到 1 层提升显著，1 层到 2 层有小幅提升，2 层到 3 层基本持平或略降。说明 2 层对于推荐场景足够了。

### 注意力可视化的发现

论文最后展示了注意力权重的可视化，揭示了一个非常关键的行为：

**在稀疏数据集上**：模型的注意力主要集中在最近的一个物品上（类似一阶马尔可夫链）。

**在稠密数据集上**：模型会关注更远的历史物品。

这直接验证了摘要中的核心论点——SASRec 通过自注意力**自适应地**在 MC 和 RNN 之间找到了平衡。

## 第五部分：总结——从这篇论文中学到什么

### 技术层面

- 为什么用自注意力而不是 RNN？（并行 + 自适应关注范围）
- 为什么需要因果 mask？（类似于机器学习的数据泄露，不能看到未来）
- 为什么共享物品嵌入？（正则化 + FFN 保证非对称性）
- 为什么不需要用户嵌入？（序列本身已隐式建模用户偏好）

### 论文写作层面

SASRec 论文有几个值得学习的写作技巧：

**动机构建**：不是"Transformer 很火所以我用它"，而是"MC 和 RNN 各有局限，自注意力恰好能解决这个矛盾"。

**与已有工作的关系**：证明自己的方法是经典方法的泛化（FMC、FPMC、FISM 都是特例），这比单纯说"我的方法更好"更有说服力。

**消融实验**：逐一去掉每个组件，证明每个设计决策都有必要。以后写论文也必须做消融实验。

**跨数据集分析**：不只是报数字，而是分析"为什么在稀疏数据集上表现出 X 行为，在稠密数据集上表现出 Y 行为"。
