# 随时记录灵感，以待探究

## 1.好的遗忘才能更好的记住?

在认知科学中，有一个现象被称为 主动遗忘（Active Forgetting） 或者更学术地说，适应性遗忘（Adaptive Forgetting）。人类大脑不是什么都记住的，它会主动抑制过时的、矛盾的、不重要的信息，从而让真正有用的记忆更容易被提取。准备考试的时候，如果脑子里塞满了无关细节，反而会干扰对核心知识的回忆。

### 第一个脉络：Machine Unlearning（机器去学习）

这是目前最热的方向，仅关于 LLM Unlearning 的综述论文就已经有多篇（其中一篇系统回顾了超过 180 篇相关论文，arxiv: 2510.25117）。它的核心问题是：如何让 LLM 定向忘掉特定信息（隐私数据、有害内容、版权内容），同时保持其他能力不受影响。动机主要来自安全和法规合规（比如欧盟 GDPR 的"被遗忘权"），但技术上的挑战和直觉是相通的——怎么精准地忘、忘了之后不崩。

### 第二个脉络：Selective Forgetting / Adaptive Forgetting

Quanta Magazine（2024年2月）和 WIRED 都报道过一项研究发现：在训练过程中主动擦除某些关键信息，反而能让模型学习新语言更快、泛化能力更强。这直接印证了"好的遗忘才能更好的记住"。arXiv 上有一篇论文叫 "Learning by Active Forgetting for Neural Networks"（arxiv: 2111.10831），提出了一种"先记住再主动遗忘（Pretraining & Purposeful Forgetting）"的训练策略，实验证明这种方法带来了自适应结构、更强的泛化能力和长期学习能力。

### 第三个脉络：认知科学的启发

"人的记忆机制和遗忘机制密不可分"，这在认知科学中有坚实的理论基础。Nature 上有研究（2018）表明，人类大脑在检索特定记忆时会主动抑制与之竞争的其他记忆（retrieval-induced forgetting），这不是大脑的缺陷，而是一种提高检索效率的适应性机制。arXiv 上有一篇综述 "Forgetting in Machine Learning and Beyond"（arxiv: 2405.20620）系统地将认知科学中的遗忘理论和机器学习中的遗忘现象做了对照分析。

### 发散思维
比如在序列推荐中：用户一年前的行为真的都值得记住吗？如果模型能"适应性地遗忘"那些已经过时的兴趣信号，只保留当前相关的行为模式，推荐效果可能反而更好。

## 2.发呆派大星会梦到海绵宝宝吗？

### 一、做梦：Sleep Consolidation → Dream Pruning

认知科学的背景是这样的：2003 年，神经科学家 Tononi 和 Cirelli 提出了**突触稳态假说（Synaptic Homeostasis Hypothesis）**。核心思想是：白天大脑在学习过程中不断增强突触连接，但这个过程不可持续——连接太多、噪声太大、能耗太高。**深度睡眠**的作用是**按比例缩小所有连接**：强连接依然较强，弱连接则衰减为零。信噪比提高了，重要的知识被保留，噪声消失。然后在 **REM 睡眠（做梦）**阶段，大脑重新激活已学到的模式并以新的方式重组，巩固有用的关联，让无关的关联衰退。

这个理论已经被直接应用到了 LLM 上。就在几天前（2026年3月初），一篇叫 **"Dream Pruning: What Happens When AI Models Sleep"** 的文章做了非常有趣的实验：

作者用 **SVD（奇异值分解）降秩** 来模拟"深度睡眠"——对 LoRA 微调后的权重矩阵做 SVD，只保留最重要的主成分方向，丢弃噪声方向。这不是随机删除权重（那相当于"脑损伤"），而是**保留知识的结构，只去掉噪声**。

实验结果很不错：经过"做梦"的 1.5B 小模型，在准确率、鲁棒性、和元认知能力上**全面超越**了没有"做梦"的 3B 大模型。作者的总结是："你不需要更大的模型，你需要更聪明的模型——而聪明来自更好的压缩，而非更多的参数。"

另外还有一篇 2024 年的论文标题直接叫 **"Dreaming is All You Need"**（arxiv: 2409.01633），探索了用睡眠巩固机制来解决持续学习中的灾难性遗忘问题。还有 OpenReview 上的 **"Learning to Self Modify and Consolidate Memories"**，提出了一种"Sleep"范式，让模型将短期脆弱记忆转化为长期稳定记忆。

### 二、发呆：Default Mode Network → LLM Daydreaming

"发呆"对应的是大脑的**默认模式网络（Default Mode Network, DMN）**。当你不在做任何具体任务、处于放空状态时，大脑并没有停止工作——DMN 会激活，做一些看似无用但实际上非常重要的事情：自由联想、回忆整合、创造性连接。

著名的研究者 Gwern 在 2025 年 7 月发表了一篇很有影响力的文章叫 **"LLM Daydreaming"**，直接追问了这个问题：为什么 LLM 拥有海量知识，却从未产生过真正的原创突破？他的回答是——因为 LLM 缺少两个关键能力：持续学习（上次我们讨论过的）和**后台思考**（发呆/做梦）。

他提出了一个具体的算法构想叫 **Day-Dreaming Loop (DDL)**：让 LLM 在空闲时持续运行一个后台进程——随机从记忆库中取出两个概念，让一个生成模型探索它们之间的非显而易见的联系，然后让一个判别模型评估结果是否有价值。有价值的发现会被存回记忆库，形成一个复利式的反馈循环。

他坦承这个过程的"命中率"会极低——绝大多数随机组合都是无用的，但这可能恰恰是创新的必要代价。人类大脑也是这样运作的：你一天中冒出无数个无关紧要的念头，但偶尔会有一个真正有价值的灵感。

### 三、把两个方向放在一起看

做梦和发呆对大脑的益处，本质上是同一件事的两个侧面：

| 人脑机制 | 功能 | LLM 对应探索 |
|---|---|---|
| 深度睡眠 | 按比例缩小突触，去噪保结构 | Dream Pruning（SVD 降秩） |
| REM 做梦 | 重新激活和重组记忆 | Self-Distillation / Replay |
| 发呆（DMN） | 自由联想、创造性连接 | Day-Dreaming Loop |

上次提出的"好的遗忘有助于记忆"，加上这次的"发呆和做梦有益"，如果再往前推一步，其实指向一个更深层的问题：**当前 LLM 的训练范式是否缺少了认知过程中那些"非生产性"但至关重要的阶段？** 当前的 LLM 只有"拼命学习"这一个状态，从不睡觉、从不发呆、从不做梦。但人类的智能恰恰依赖这些"非生产性"的过程。

### 四、和推荐系统的潜在联系

在序列推荐中，用户的行为序列就像一段"清醒时的记忆"。模型在处理这段序列时，是否也可以受益于类似"做梦"的机制？比如：在训练时引入一个"巩固阶段"，让模型重新审视用户的历史行为，通过某种压缩或重组来强化重要模式、遗忘噪声交互？或者在推理时，对长序列做类似 SVD 的"信息蒸馏"，保留核心兴趣信号、去除随机点击噪声？

### 五、小结

目前已经积累了三个相关的直觉：好的遗忘有助于记忆、做梦有助于知识巩固、发呆有助于创造性连接。它们指向同一个底层主题——**认知启发的 AI 机制设计**。这在学术上是一个有故事、有深度的方向，等我基础打扎实后，可以认真考虑是否作为论文选题的一个切入角度。
