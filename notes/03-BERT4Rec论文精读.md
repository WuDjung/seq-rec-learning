

# BERT4Rec 论文精读

**论文标题**: BERT4Rec: Sequential Recommendation with Bidirectional Encoder Representations from Transformer

**作者**: Fei Sun 等 (Alibaba Group)

**发表**: CIKM 2019

---

## 核心直觉：理解 BERT4Rec 和 SASRec 的区别

不妨用考试来类比：

**SASRec（单向）**像是一场"接龙游戏"——给你前面几个词，预测下一个词。你只能看到左边的内容。

**BERT4Rec（双向）**像是一场"完形填空"——一段话中间挖了几个空，让你根据**上下文**来填。你能同时看到左边和右边的内容。

核心差异就是这一点。接下来我们逐部分展开。

---

## 第一部分：论文的动机——SASRec 有什么不足？

BERT4Rec 的作者认为 SASRec 有两个局限：

**局限一：单向架构限制了表示能力。** 在 SASRec 中，位置 $t$ 的隐藏表示只能编码位置 $1$ 到 $t$ 的信息。但实际上，位置 $t$ 后面的物品也可能提供有用的上下文。比如用户的序列是"手机 → 手机壳 → 贴膜 → 充电器"，如果我要理解"手机壳"这个行为的含义，看到后面的"贴膜"和"充电器"会帮助我确认——这个用户确实在为新手机购买配件。

**局限二：用户行为不总是严格有序的。** 用户可能在同一天内浏览了多个物品，这些物品之间并没有严格的因果顺序。强制假设严格的从左到右顺序可能引入噪声。

### 批判性思考

这两个论点合理但也不是无懈可击。SASRec 的作者后来也有回应：在预测"下一个物品"这个任务中，你本来就不应该看到未来的信息，因为推理时确实没有未来可看。BERT4Rec 的训练（双向看上下文）和推理（只有左侧上下文 + 末尾一个 mask token）之间存在**训练-推理不一致**的问题。这是 BERT4Rec 最受争议的地方，我们后面会详细讨论。

## 第二部分：核心方法

### 2.1 模型架构

BERT4Rec 的 Transformer 架构和 SASRec **几乎完全相同**——同样是物品嵌入 + 位置嵌入 → 多层 Transformer Encoder → 内积预测。唯一的关键区别是：

| | SASRec | BERT4Rec |
|---|---|---|
| 注意力方向 | 单向（因果 mask） | 双向（无因果 mask） |
| 训练任务 | Next-item prediction | Masked item prediction (Cloze) |
| 推理方式 | 取最后一个位置的输出 | 在末尾加 [mask] token |
| 对应的 NLP 模型 | GPT | BERT |

### 2.2 训练任务：Masked Item Prediction（完形填空）

这是 BERT4Rec 最核心的创新。训练时，随机将序列中一定比例（$\rho$）的物品替换为特殊的 `[mask]` token，然后让模型预测被遮盖的物品是什么。

```
原始序列:  [手机, 手机壳, 贴膜, 充电器, 耳机]
遮盖后:    [手机, [mask], 贴膜, [mask], 耳机]
预测目标:   mask1 = 手机壳, mask2 = 充电器
```

模型在预测 `mask1` 时，可以同时看到左边的"手机"和右边的"贴膜、耳机"；预测 `mask2` 时，可以同时看到左边的"手机、贴膜"和右边的"耳机"。

损失函数是标准的 Cross-Entropy，只在 `[mask]` 位置计算：

$$\mathcal{L} = \frac{1}{|S_u^m|} \sum_{v_m \in S_u^m} -\log P(v_m = v_m^* \mid S_u')$$

其中 $S_u^m$ 是被遮盖的物品集合，$S_u'$ 是遮盖后的序列。

### 2.3 推理方式

推理时没有"未来"可以看，所以 BERT4Rec 的做法是在序列末尾追加一个 `[mask]` token：

```
输入:  [手机, 手机壳, 贴膜, 充电器, [mask]]
预测:  [mask] 位置的输出 → 对所有物品打分 → 推荐 Top-K
```

这个 `[mask]` 位置可以通过双向注意力看到左边所有的历史物品，然后预测下一个。

### 2.4 Mask 比例 $\rho$ 的选择

这是一个关键的超参数。论文在不同数据集上调出了不同的最优值：

| 数据集 | 稀疏程度 | 最优 mask 比例 $\rho$ |
|---|---|---|
| Beauty | 非常稀疏 | 0.6 |
| Steam | 中等 | 0.4 |
| ML-1m | 稠密 | 0.2 |
| ML-20m | 稠密 | 0.2 |

规律是：**数据越稀疏，mask 比例越高**。直觉上，稀疏数据集的序列很短，如果 mask 比例太低，每条序列几乎没有被遮盖的物品，训练信号太少。

## 第三部分：和 SASRec 的深入对比

### 3.1 训练样本数量

这是 BERT4Rec 的一个重要优势。对于长度为 $n$ 的序列：

**SASRec** 产生 $n$ 个训练样本（每个位置预测下一个）。

**BERT4Rec** 在每个 epoch 中随机 mask，多个 epoch 可以产生 $\binom{n}{k}$ 种不同的 mask 组合（$k$ 是每次 mask 的物品数）。这意味着 BERT4Rec 的有效训练样本量远大于 SASRec。

### 3.2 训练-推理一致性

这是 BERT4Rec 的核心争议点：

**SASRec**：训练时预测下一个物品，推理时也预测下一个物品——完全一致。

**BERT4Rec**：训练时填补序列中间的空缺（双向上下文），推理时预测序列末尾的下一个物品（实际上只有左侧上下文有意义）——存在不一致。

论文作者通过在训练时也加入一些"只 mask 最后一个物品"的样本来缓解这个问题，相当于做了一定程度的"微调"来对齐推理场景。

### 3.3 实际效果对比

论文中 BERT4Rec 在所有数据集上都超过了 SASRec，在 ML-1m 上的提升尤为明显（HR@10: 0.697 vs 0.663）。但后来有一些复现研究对这个结论提出了质疑——在某些设置下 SASRec 经过仔细调参后可以追平甚至超过 BERT4Rec。这说明两种方法各有优劣，没有绝对的赢家。

## 第四部分：实验细节

### 超参数对比

| 超参数 | SASRec | BERT4Rec |
|---|---|---|
| 层数 | 2 | 2 |
| 注意力头数 | 1 | 2 |
| 优化器 | Adam, lr=1e-3 | Adam, lr=1e-4 |
| Dropout | 0.2 (ML-1m) | 0.2 (ML-1m) |
| Max length | 200 (ML-1m) | 200 (ML-1m) |
| 损失函数 | BCE + 负采样 | Cross-Entropy |

注意 BERT4Rec 的学习率（1e-4）比 SASRec（1e-3）小了一个数量级，而且使用了 weight decay 和 gradient clipping。这些都是更精细的训练技巧。

### 消融实验关键发现

论文做了几个重要的消融：

**单向 vs 双向**：在 BERT4Rec 上加因果 mask（变成单向），效果下降。说明双向确实有帮助。

**Mask 比例的影响**：太高或太低都不好。太低训练信号不足，太高上下文信息太少。

**层数的影响**：和 SASRec 类似，2 层效果最好，3 层开始下降。

## 第五部分：总结——从这篇论文中学到什么

### 技术层面

- BERT4Rec 和 SASRec 的核心区别是什么？（双向 vs 单向、Cloze vs Next-item）。

- 为什么双向模型不能直接用 Next-item prediction 训练？（信息泄露——每个位置都能间接看到目标物品）。

- BERT4Rec 的训练-推理不一致问题是什么？如何缓解？(BERT4Rec 的训练-推理不一致问题本质是：训练时 mask 在序列中间（能看到两侧），推理时 mask 在序列末尾（只能看到左侧），模型面对的输入条件不同。 通过在训练中混入"只 mask 末尾"的样本可以缓解，但无法完全消除。这也是为什么后续有些研究发现 SASRec 经过仔细调参后可以追平 BERT4Rec——单向模型虽然信息量少，但训练-推理一致性更好。)

### 论文写作层面

BERT4Rec 的论文写作有一个值得学习的技巧——**和 SASRec 的逐点对比**。整篇论文始终把 SASRec 作为核心参照对象，每介绍一个设计决策都说清楚"SASRec 是这样做的，我们为什么不同"。这种对比式写法非常有效，你以后写论文时如果是在某个方法的基础上改进，可以用同样的策略。
